Существующие и потенциальные проблемные места:
    1. единый api для внутренних пользователей (покупателей) и внешних (b2b) на MES
    2. отсутствует масштабирование сервисов
    3. нет никакой информации о саге, даже в процессах, которые потенциально несут угрозу ошибок (длительные операции расчета и оповещения клиента), отсутствуют механизмы оповещения пользователей об ошибках в приеме заказа в работу
    4. нет гетвеев, для маршрутизации запросов внутри системы
    5. отсутствие кэширования для информации, которая не изменилась/статична
    6. отсутствие сервиса оповещений, для информирования пользователей
    7. задержки в поставке на пром нового функционала из-ща долгого ручного тестирования и отсутствия автоматизации
    8. отсутствие логирования и/или трейсинга, мониторинга

Инициативы, которые необходимы для устранения нежелательных ситуаций
    1. создание отдельной точки входы для b2b клиентов
    2. внедрение гетвея для маршрутизации запросов
    3. масштабирование микросервисов до нескольких инстансов
    4. создание кэширования для MES сервиса
    5. создание сервиса оповещений
    6. внедрение саги для процессов ведения заказа от момента создания до принятия в работу, чтобы в случае необходимости можно было откатить процесс и оповестить пользователя
    7. создание систем мониторинга, логирования, трейсинга 
    8. автоматизация тестирования, введение юнит тестов

Система через полгода:Система обладает автоматическим масштабирование для работы с возрастающей нагрузкой и механизмами для борьбы с ошибочными (незавершенными транзакциями). Основная масса запросов обрабатывается быстрее и запросы внешних систем и клиентов из магазина не пересекаются, что уменьшает количество ошибок при формировании заказов. Автотесты ускоряют поставку продукта в пром

Приоритет задач и три основных инициативы:
    1. реализация гетвея для маршрутизации запросов - сможем подготовить систему к масштабированию каждого из микросервисов, настроем корректное движение запросов, выделим основные ошибки, которые стопорят процессы. Временно решим проблему с внешними запросами напрямую в MES систему (позже внедрим отдельный сервис для запросов из вне)
    2. подготовка сервисов к масштабированию: дополнительные инстансы при возрастающей нагрузке на MES и CRM. (при подготовке сервисов к масштабированию написание юнит тестов и автоматизированных тестов) 
    3. реализовать сагу для уменьшение заказов, которые ожидают пользователи, но которые никогда не будут сделаны
